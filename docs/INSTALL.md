## Update model

The SLOP install directory (`/opt/mediastack`) is an **HTTPS git clone** of
`Nnyan/SLOP`, owned by the **service user `mediastack`**. Updates pull from
`origin/main` using `ms-update` or `deploy.sh --update`; there is no manual
`scp`/`rsync` copy step.

```bash
# Standard update (recommended)
sudo ms-update

# Alternative: use the deploy script
sudo -u mediastack /opt/mediastack/deploy.sh --update
```

Both tools:
1. Run all file-touching ops (`git fetch`, `pip install`, `npm ci && npm run build`)
   as the service user (`sudo -u mediastack ...`).
2. Perform a `fetch` + fast-forward attempt; fall back to `reset --hard origin/main`
   automatically if the clone is diverged.
3. Rebuild the frontend (`backend/static/` is gitignored and deleted by `reset --hard`;
   the build always runs when frontend files changed).
4. Restart the service with `systemctl restart mediastack` (root-only op).

For the full ownership model, shared helper functions (`tools/deploy_lib.sh`), and a
manual recovery runbook for diverged clones, see [DEPLOY.md](DEPLOY.md).

## Service port

The canonical service-port environment variable is **`MS_PORT`** (default `8080`).

`deploy.sh` writes `MS_PORT=<value>` into `/opt/mediastack/.env` and bakes the port
into the generated systemd unit. To change the port:

```bash
# Edit .env
sed -i 's/^MS_PORT=.*/MS_PORT=9090/' /opt/mediastack/.env
# Regenerate the unit with the new port (or edit /etc/systemd/system/mediastack.service directly)
systemctl restart mediastack
```

The legacy variable name `MEDIASTACK_PORT` is read as a deprecated fallback; a warning
is printed when the fallback fires. New installs use `MS_PORT` only.

## Operator environment

Operator env `MS_TRUSTED_HOSTS` / `DOMAIN` is authoritative in the install-dir `.env`
(`/opt/mediastack/.env`); the canonical edit point is: edit `.env` then
`systemctl restart mediastack`.

The systemd unit generated by `deploy.sh` carries `EnvironmentFile=/opt/mediastack/.env`,
which injects every `KEY=VALUE` into the process environment. `MS_TRUSTED_HOSTS` and
`DOMAIN` are read via `os.environ` in the FastAPI startup code, so values in `.env`
take effect on the next `systemctl restart mediastack`.

```bash
# Example: trust a LAN hostname
echo 'MS_TRUSTED_HOSTS=myserver.local,10.0.1.51' >> /opt/mediastack/.env
systemctl restart mediastack
```
