## How to add to this file

When you discover floating work — in a Robot retro, a code review, a stale
doc, an observation file — append a line to the appropriate section above
with: status tag, [category], description, source, date.

When you SCHEDULE an open item into a wave: change `[ ]` to `[→ S-NN-stream]`
and link the wave file.

When you COMPLETE an item: change to `[x]` and move to the Done section.
Prune Done items > 60 days old in the next monthly sweep.

When you decide NOT to fix: change to `[—]` and put a one-line reason.

The Done section is intentionally bounded — it's not a permanent archive,
it's a recent-history reference. The full history lives in git + the
`.claude/run-archive/` tree.
